# Qwythos-Ascend 故障排查指南

## 1. API 服务无法启动

**现象**: `bash start_api.sh start` 启动失败或 `curl /health` 无响应

**排查步骤**:
```bash
# 1. 检查 NPU 驱动
npu-smi info
# 如果无输出: 驱动异常

# 2. 检查环境变量
echo $LD_LIBRARY_PATH | grep ascend
# 应包含 /usr/local/Ascend/... 路径

# 3. 查看日志
tail -50 /root/qwythos_server.log

# 4. 检查是否端口被占
ss -tlnp | grep 8000
fuser -k 8000/tcp  # 强制释放
```

**常见原因**:
- `LD_LIBRARY_PATH` 未设置 → 执行 `source /etc/profile.d/ascend.sh`
- `libascendcl.so` 找不到 → 检查 CANN 7.0.0 安装
- 端口占用 → `bash start_api.sh restart`
- 模型加载超时 → 等待2分钟初始化

## 2. NPU 算子执行失败

**现象**: `RuntimeError: xxx: 507011` 或 `500002`

| 错误码 | 含义 | 解决 |
|:-----|:----|:----|
| 507011 | 跨芯片显存访问 | 检查aclrtSetDevice是否在malloc/h2d前调用 |
| 500002 | GE 运行错误 | 算子模型损坏，重新 ATC 编译 |
| 145001 | 运行时错误 | 检查 .om 模型路径 |
| 145003 | 执行失败 | 算子未正确加载，重启服务器 |
| 100024 | 算子未注册 | 检查 SOC_VERSION 设置 |

**解决**:
```bash
# 环境变量
source /usr/local/Ascend/ascend-toolkit/7.0.0/x86_64-linux/script/set_env.sh
export SOC_VERSION=Ascend310

# 重新编译算子
atc --model=om_models/xxx.onnx --framework=5 \
    --output=om_models/xxx --soc_version=Ascend310 \
    --input_shape="A:1,4096;B:4096,4096" --log=error
```

**关键: 设备上下文** — Ascend 310的 `aclrtSetDevice` 必须在以下操作前调用:
- `aclrtMalloc` — 显存分配
- `aclrtMemcpy` (H2D) — 主机到设备拷贝
- `aclrtMemcpy` (D2D) — 设备到设备拷贝
- `aclmdlLoadFromFile` — 加载算子模型
- `aclmdlExecute` — 执行推理

## 3. 模型输出乱码

**现象**: 返回的文字是乱码/多语言混杂（如"小姑娘pragmamerprioelves培养出 områdetorten"）

**原因**:
1. **线性注意力未实现** — 24层 Linear Attention 使用简单投影代替 DeltaNet SSM
2. **Final RMSNorm 缺失** — 32层后需要 `model.norm.weight` 归一化
3. **Chat template 问题** — 需要 `trust_remote_code=True` 加载 Qwen3.5 tokenizer

**排查**:
```bash
# 检查最终归一化
python3 -c "
import sys; sys.path.insert(0,'/root/qwythos_engine')
from engine.weights import WeightLoader
wl = WeightLoader('/root/models/Qwythos-9B-Claude-Mythos-5-1M')
cw = wl.load_all()
print('norm:', cw.get('model.norm.weight').shape if 'model.norm.weight' in cw else 'MISSING')
print('lm_head:', cw.get('lm_head.weight').shape if 'lm_head.weight' in cw else 'MISSING')
"
```

## 4. 芯片显存不足

**现象**: `malloc 返回 0`, `507011`, 或 `npu-smi info` 显示接近 7759MB

**原因**:
- 单芯片 8GB 限制
- v8 代码每层 400MB × 8 层 = 3.2GB
- 服务器进程 + 权重 = ~6GB/chip 0

**解决**:
```bash
# 1. 查看当前显存
npu-smi info

# 2. 清理残留进程
bash /root/qwythos_engine/start_api.sh stop
pkill -9 -f npu_server
sleep 5  # 等待驱动释放

# 3. 重新启动
bash /root/qwythos_engine/start_api.sh start

# 4. 减少上下文长度
# 修改 MAX_CONTEXT = 4096 (server.py中)
```

## 5. ATC 编译失败

**现象**: `ATC run failed`

**常见原因**:
```bash
# 1. SOC_VERSION 未设置
export SOC_VERSION=Ascend310

# 2. ASCEND_OPP_PATH 错误
export ASCEND_OPP_PATH=/usr/local/Ascend/ascend-toolkit/7.0.0/opp

# 3. PYTHONPATH 缺少 tbe
export PYTHONPATH=/usr/local/Ascend/ascend-toolkit/7.0.0/python/site-packages:\
  /usr/local/Ascend/ascend-toolkit/7.0.0/opp/built-in/op_impl/ai_core/tbe:$PYTHONPATH

# 4. 输入形状不匹配
# ONNX 模型输入为 [1,4096]，input_shape 必须一致
atc --input_shape="A:1,4096;B:4096,4096"

# 5. Ascend 310 不支持 FP16×FP16→FP16
# 必须在ONNX中插入Cast到FP32输出
```

## 6. API 返回 Internal Server Error

**现象**: curl 返回 500，日志显示概率 NaN

**原因**:
- Logits 经 Top-K 过滤后全部为 -inf
- Softmax 产生 NaN 概率

**解决**:
```bash
# 更新 npu_server.py 到 v2.0+（已包含数值稳定采样）
# 或调低 temperature 到 0.1
# 或调大 top_k 到 100
```

## 7. 交叉编译环境恢复

如果环境变量丢失：
```bash
source /usr/local/Ascend/ascend-toolkit/7.0.0/x86_64-linux/script/set_env.sh
export SOC_VERSION=Ascend310
export LD_LIBRARY_PATH=/usr/local/Ascend/ascend-toolkit/7.0.0/lib64:\

/usr/local/Ascend/driver/lib64:\
/usr/local/Ascend/driver/lib64/common:\
/usr/local/Ascend/driver/lib64/driver
```

## 8. GitHub Push 失败

**现象**: `Could not connect to github.com:443`

**原因**: 服务器/本地网络限制

**解决**:
```bash
# 检查连接
curl -v https://github.com 2>&1 | head -20

# 使用代理（如适用）
git config --global http.proxy http://proxy:port
git config --global https.proxy http://proxy:port

# 或使用 SSH 方式
git remote set-url origin git@github.com:ziphao2010/qwythos-ascend.git
```
