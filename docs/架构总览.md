# Qwythos-Ascend 架构总览

## 整体架构

```
用户请求 → FastAPI → Python Engine → ACL C API → Ascend 310 NPU
                          ↕
                    ATC编译的 .om 算子
                          ↕
                    4芯片 × 8GB HBM
```

## 推理管线

```
输入文本 → Tokenizer → Embed → 32层 NPU 前向 → LM Head → Sample → Token
                                  ↕
    每层: RMSNorm → (QKV投影 → 注意力 → 输出投影 → 残差) → RMSNorm → MLP → 残差
    24层 DeltaNet (线性注意力) + 8层 Full Attention (GQA)
```

## 4芯片分布

```
Chip 0 (8000:08:00.0): Layers 0-7    ~3.2GB
Chip 1 (8000:09:00.0): Layers 8-15   ~3.2GB
Chip 2 (8000:0A:00.0): Layers 16-23  ~3.2GB
Chip 3 (8000:0B:00.0): Layers 24-31  ~3.2GB
```

## 模型架构

```
Qwythos-9B (基于 Qwen3.5-9B)
├── 文本解码器: 32层, 4096 hidden, 16Q/4KV heads
├── 视觉编码器: 27层 ViT, 1152 hidden (冻结)
├── 词汇表: 248,320 tokens
├── 上下文: 1M tokens (YaRN RoPE)
└── 权重: 760张量, 18.8GB FP16
```

## 14个 NPU 算子

| 算子 | ONNX 定义 | 用途 |
|:----|:---------|:-----|
| mm_1_4096_4096 | [1,4096]×[4096,4096] | Q/O/MLP 投影 |
| mm_1_4096_1024 | [1,4096]×[4096,1024] | K/V 投影 |
| mm_1_4096_12288 | [1,4096]×[4096,12288] | MLP gate/up |
| mm_1_12288_4096 | [1,12288]×[12288,4096] | MLP down (保留) |
| mm_1_6144_4096 | [1,6144]×[6144,4096] | MLP down (拆分) |
| mm_1_1024_4096 | [1,1024]×[1024,4096] | KV→输出 |
| mm_1_4096_8192 | [1,4096]×[4096,8192] | QKV 合并 |
| ops_rmsnorm | Pow+ReduceMean+Sqrt+Div | 层归一化 |
| ops_silu | Sigmoid+Mul | 激活函数 |
| ops_softmax | Softmax | 注意力权重 |
| ops_add | Add | 残差连接 |
| ops_mul | Mul | 逐元素乘 |
| fused_attn | MatMul+Transpose+Softmax+MatMul | 全注意力 |

## 优化历程

```
v5  (基础)   19s    ← 单芯片，每层权重 malloc/h2d/free
v8  (缓存)   0.4s   ← 权重缓存，8层命中 17×加速
v9  (融合)   +      ← 融合注意力 (0.3ms NPU)
v10 (KVC)    15s    ← KV Cache 多token注意力
v11 (4芯片)  2.2s   ← 4×8层分布 + 预加载 ← 当前最新
v12 (流水线) ~2.2s   ← 流水线并行 (吞吐4×) ← 开发中
```
