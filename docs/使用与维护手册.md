## 🚀 日常使用

### 启动 API 服务器

```bash
ssh root@192.168.1.199
cd /root/qwythos_engine
bash start_api.sh start    # 启动
bash start_api.sh stop     # 停止
bash start_api.sh restart  # 重启
bash start_api.sh status   # 查看状态
```

服务器约 **2分钟** 初始化（加载18GB权重）。启动后自动运行在 `0.0.0.0:8000`。

### 调用 API

```bash
# 健康检查
curl http://192.168.1.199:8000/health

# 对话 (Chat Completions)
curl -X POST http://192.168.1.199:8000/v1/chat/completions \
  -H "Authorization: Bearer wsh101007" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwythos-9b","messages":[{"role":"user","content":"你好"}],"max_tokens":64}'

# 流式对话
curl -X POST http://192.168.1.199:8000/v1/chat/completions \
  -H "Authorization: Bearer wsh101007" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwythos-9b","messages":[{"role":"user","content":"讲个故事"}],"max_tokens":256,"stream":true}'

# 文本补全
curl -X POST http://192.168.1.199:8000/v1/completions \
  -H "Authorization: Bearer wsh101007" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwythos-9b","prompt":"Once upon a time","max_tokens":64}'

# 列表模型
curl -X POST http://192.168.1.199:8000/v1/models \
  -H "Authorization: Bearer wsh101007"
```


## 🐛 已知问题与限制

1. **4芯片 D2D copy** — PM8532 PCIe交换机延迟 ~50ms/次
2. **Attention在CPU** — 8个全注意力层在CPU计算，GPU版需要fused_attn.om
3. **线性注意力占位** — 24层Linear Attention使用简单投影，未实现DeltaNet SSM
4. **模型输出质量** — 因线性注意力占位，输出为随机词汇混杂
5. **启动慢** — 加载18GB safetensor需2分钟
6. **8K上下文限制** — NPU显存限制KV Cache大小，1M上下文待优化

## 📊 推理时间估算

```
Prompt N tokens + Generate M tokens ≈ (N+M) × 2.2s
例: prompt=100, generate=50 → 150 × 2.2s = 330s ≈ 5.5分钟
```
